# Optimization Engine: Iterate, Bundle, or Kill

## Purpose
A strict, emotionless decision tree for managing listings after 14 days of live data.

## The 14-Day Decision Tree
1. **Views < 50**: 
   - *Diagnosis*: SEO failure. The listing is invisible.
   - *Action*: Run `COMPETITOR_LISTING_TEARDOWN.md`. Rewrite Title and all 13 Tags. Change Image 1 hook.
2. **Views > 100, but Favorites < 5**: 
   - *Diagnosis*: Click-through rate (CTR) is okay, but the listing page is repelling buyers.
   - *Action*: Overhaul Images 2-4. Ensure the "Problem/Solution" is instantly clear. Check pricing.
3. **Favorites > 10, but Orders = 0**: 
   - *Diagnosis*: High desire, high friction. 
   - *Action*: Add a limited-time "Launch Discount" (e.g., 20% off for 48 hours). Add a FAQ section addressing the #1 buying objection.
4. **Conversion Rate > 3%**: 
   - *Diagnosis*: WINNER.
   - *Action*: Do not touch the SEO. Immediately create a complementary product to bundle with this winner. Increase Pinterest pinning frequency for this specific listing.
